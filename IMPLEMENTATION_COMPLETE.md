## 📋 IMPLEMENTATION DETAILS

### OAuth 2.0 Flow
```
1. User clicks Login
2. Redirect to Salesforce OAuth URL
3. User authorizes in Salesforce
4. Salesforce redirects to callback with code
5. App exchanges code for access token
6. Token stored in localStorage
7. App is now authenticated
```

### Fetch Validation Rules
```
1. App uses Tooling API
2. Queries ValidationRule metadata type
3. Filters for Account object rules
4. Returns rule details (name, status, etc.)
5. Displays in interactive table
```

### Toggle Rule Status
```
1. User clicks toggle button
2. App makes PATCH request to Tooling API
3. Updates rule Active status
4. Salesforce confirms change
5. UI updates immediately
```

### Deploy Changes
```
1. User clicks Deploy button
2. App uses Metadata API
3. Creates deployment request
4. Salesforce processes deployment
5. App monitors status
6. Shows completion message
```
